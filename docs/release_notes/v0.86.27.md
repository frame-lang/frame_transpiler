# v0.86.27 — CLI Module Compile Restored, Debug Artifacts v2 (Py/TS)

Release date: 2025-11-14

Highlights
- Restored full module compile via main CLI path for Python/TypeScript.
  - `framec -l python_3|typescript --emit-debug path/to/module.frm > out.(py|ts)`
  - Auto-detects `@target` in file and routes to the V3 module compiler (no longer single-body only).
- Debug artifacts upgraded for Py/TS module compiles:
  - visitor-map v2: includes targetColumn/sourceColumn (schemaVersion=2)
  - debug-manifest v2: includes handlers with compiled IDs and param names (schemaVersion=2)
- Tests expanded:
  - Added Py/TS debugger fixtures (multi-state, multi-handler)
  - Rust: added curated exec positive and converted forward-without-parent to negatives (E403)
- Bug #059 closed: CLI now supports module compile with `--emit-debug` trailers.

Details
- CLI (non-demo compile path)
  - Auto-routes files containing `@target` to the module compiler in `run_file`/`run_file_debug`.
  - Preserves single-body compile for body-only inputs.
- Compiler (Py/TS)
  - Module path emits visitor-map with columns and debug-manifest including handlers.
- Tests/Fixtures
  - v3_debugger (Py/TS): multi_state_multi_handler.
  - Rust v3_control_flow: inline_after_nonterminal_then_transition_exec (positive), plus E403 negatives.

Docs
- Updated docs/README.md and architecture_v3/README.md with “How To: Compile Modules (CLI)”
- Debugger integration doc implicitly covered via trailers; visitor-map/debug-manifest schema updates reflected.
- Closed Bug #059 in docs/bugs.

Upgrade Notes
- No breaking changes; visitors for other languages unaffected.
- If asserting visitor-map payload shape in external tools, note the new `targetColumn` and `sourceColumn` fields and `schemaVersion: 2`.

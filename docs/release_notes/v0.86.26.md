# v0.86.26 — Debugger Artifacts Ready (Py/TS), Runner Upgrades, Rust Exec Breadth

Release date: 2025-11-14

Highlights
- Python and TypeScript are debugger-ready with stable artifact emission:
  - errors-json (structured compiler diagnostics, schemaVersion)
  - frame-map (origin map: frame/native)
  - visitor-map (module path, Py/TS) with targetLine/sourceLine/origin
  - debug-manifest (system, states, compiled state IDs)
- Test runner adds `v3_debugger` category and validates trailer presence and shape.
- Curated exec suites (Py/TS/Rust) passing for seeded categories; full Py/TS V3 suites are green.
- Bugs triage: Bug #058 closed as “Won’t Fix (Superseded)” in favor of standardized trailers/sidecars.

Details
- Compiler/CLI
  - `--emit-debug` enables all debug trailers; env flags supported: `FRAME_ERROR_JSON`, `FRAME_MAP_TRAILER`, `FRAME_DEBUG_MANIFEST`, and optional `FRAME_NATIVE_SYMBOL_SNAPSHOT`.
  - Debug-manifest includes `schemaVersion`, `system`, `states`, and compiled state IDs used by expanders.
- Runner
  - Recognizes `v3_debugger` tests; sets env flags and asserts visitor-map + debug-manifest.
  - Continues to extract sidecars: `.frame-map.json`, `.visitor-map.json`, `.debug-manifest.json`.
- Tests
  - Python: All V3 suites and curated exec pass.
  - TypeScript: All V3 suites and curated exec pass.
  - Rust: Seeded curated exec and V3 suites pass.

Docs
- Added “How To: Collect Debugging Artifacts (Py/TS)” to `docs/README.md` and `docs/framepiler_design/architecture_v3/README.md`.
- Updated bugs index and moved #058 to closed.

Upgrade Notes
- No breaking changes to CLI or codegen.
- Debugger teams can consume artifacts by compiling module files with `--emit-debug` and reading the sidecars.

# Frame Transpiler v0.86.49

- Aligns the TypeScript runtime type declarations (`frame_runtime_ts/index.d.ts`) with the V3 generator and runtime implementation:
  - `FrameEvent` now exposes a `message` property and constructor signature that match the runtime.
  - `FrameCompartment`'s constructor and fields (`enterArgs`, `exitArgs`, `stateArgs`, `forwardEvent`, etc.) are declared so generated code compiles cleanly with `tsc`.
- Fixes three TypeScript CLI bugs related to runtime types and module validation:
  - **Bug #078** — TS runtime d.ts mismatch with generator calls (constructor arity and missing fields) is resolved by the new `.d.ts`.
  - **Bug #073** — TypeScript generator emits duplicate class methods per state (reopened via #078) is confirmed fixed; the CLI emitter still generates a single public method per interface and now passes minimal module validation with the corrected types.
  - **Bug #074** — TypeScript generator omits actions/domain declarations and drops interface params (reopened via #078) is confirmed fixed; CLI fixtures emit the correct fields and compile successfully.
- Keeps the V3 architecture and MIR semantics unchanged; this is a focused bug‑fix release to restore TypeScript CLI compile‑time confidence for runtime‑backed modules.

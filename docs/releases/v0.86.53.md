# Frame Transpiler v0.86.53

- Fixes **Bug #082** by updating V3 TypeScript interface wrappers and the
  router so that interface methods (including `drainCommands`) return whatever
  their Frame bodies return, rather than always yielding `undefined`.
- Closes **Bug #083** from the compiler perspective, confirming that `stopped`
  events routed via `runtimeMessage(payload)` correctly set `isPaused === true`
  in the generated TS and that prior failures were due to TypeScript
  environment setup in external validators.
- Keeps all V3 TypeScript CLI fixtures passing, with updated expectations for
  the public wrapper signatures (`start(): any`, `drainCommands(): any`, etc.).

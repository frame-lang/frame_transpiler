# Frame Transpiler v0.86.46

- Aligns the V3 Python runtime with the documented system/state parameter semantics:
  - `compile_module_demo` now parses `system` parameter lists (`$(start)`, `$>(enter)`, `domain`), partitions constructor arguments into `startArgs`, `enterArgs`, and `domainArgs`, and seeds the initial `FrameCompartment` for the first declared state (via Arcanum).
  - The Python kernel fires `$enter` on construction and on `_frame_transition`, matching the abstract runtime model.
- Introduces a V3 Python TrafficLight runtime fixture:
  - `framec_tests/language_specific/python/v3_systems_runtime/positive/traffic_light_system_exec.frm` exercises system/state parameters and runtime transitions under the new kernel.
- Updates the Python router and interface dispatch model:
  - Event-specific handlers are emitted as `_event_<name>(self, __e, compartment)`, and `_frame_router` invokes them using the event’s message.
  - Public interface methods (e.g., `tick()`) now construct `FrameEvent` instances and delegate to `_frame_router`, so users call `tl.tick()` as in the v2 grammar.
- Adjusts the V3 docs and capability matrix:
  - Marks system parameters as structurally complete with Python runtime coverage (TrafficLight) and clarifies that TS remains compile-only for this feature.
  - Documents the current status of `system.return` as placement-enforced (E407) with runtime behavior inherited from the v2 model pending additional V3 exec fixtures.

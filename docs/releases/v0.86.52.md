# Frame Transpiler v0.86.52

- Clarifies the resolution of **Bug #081** by documenting that V3 Frame
  statements (including transitions `-> $State`) are strictly
  start‑of‑line (SOL) anchored and that inline forms such as
  `this.lifecycle = "terminated"; -> $Terminated;` are out of spec and
  will not be rewritten into `_frame_transition` calls.
- Confirms that the canonical adapter semantics live in the in‑repo minimal
  AdapterProtocol fixture and the shared `framepiler_test_env/adapter_protocol`
  Node harness, and that external validators must update their FRM to follow
  these rules rather than relying on inline `->` syntax.
- Contains no compiler code changes relative to v0.86.51; this is a
  documentation and bug‑tracking alignment release.

# Frame Transpiler v0.86.50

- Adds public interface wrappers and a functional `_frame_router` for V3 TypeScript systems so generated modules can be used directly from TypeScript without passing `FrameEvent`/`FrameCompartment` objects.
- Fixes **Bug #079** by:
  - Emitting `public` methods for interface events (e.g., `start()`, `runtimeMessage(payload)`), which construct `FrameEvent` instances internally and call `_frame_router`.
  - Keeping handler-style methods private (e.g., `_event_start`, `_event_runtimeMessage`) that switch on `c.state` and execute spliced state bodies.
- Keeps all existing V3 TypeScript CLI fixtures green under `@tsc-compile`, confirming that the new API surface is type-safe and compatible with the `frame_runtime_ts` package.

# Frame Transpiler v0.86.54

- Implements full V3 `system.return` semantics for Python and TypeScript module targets:
  - Interface methods now allocate a per-call return slot backed by a `_system_return_stack` and always return the final `system.return` value (or the interface header default).
  - Event handlers may declare header initializers (`handler(a, b) = Expr`) and use `return expr` sugar, which is expanded to `system.return = expr; return`.
  - Actions and non-static operations can read and assign `system.return`, participating in the same per-call return value.
- Updates the V3 validator, grammar, and capability matrix to reflect the new policy (no E407 restriction on operations) and documents header-based initialization in the V3 grammar and runtime docs.
- Extends the V3 capability suites for Python and TypeScript with new fixtures exercising `system.return` behavior in handlers, actions, and operations, keeping all V3 transpile-only tests passing.
- Addresses **Bug #081** by ensuring AdapterProtocol-style fixtures compiled through the V3 TypeScript path use the per-call `system.return` stack in public interface wrappers, aligning CLI behavior with the shared adapter test environment.

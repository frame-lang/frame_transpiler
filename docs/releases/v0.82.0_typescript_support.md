# Frame v0.82.0 Release Notes - TypeScript Support

## Release Date: 2025-10-13

## Overview
Frame v0.82.0 introduces TypeScript as a new target language for the Frame transpiler, enabling developers to generate type-safe, event-driven state machines in TypeScript. This release provides comprehensive support for Frame's state machine semantics while leveraging TypeScript's type system and modern JavaScript features.

## Major Features

### TypeScript Code Generation
The Frame transpiler now supports full TypeScript code generation with:
- Complete state machine implementation with runtime support
- Event-driven architecture with FrameEvent and FrameCompartment classes
- State dispatchers with proper event routing
- Interface method generation with return value handling
- Event handler generation for all states
- Actions transpiled as private class methods
- Domain variables as TypeScript class properties

### TypeScript State Machine Features
- **State Transitions**: Fully functional `_frame_transition()` method
- **Enter/Exit Handlers**: Support for `$>` and `$<` events
- **Event Routing**: Dispatcher pattern for efficient event handling
- **Hierarchical States**: Foundation for hierarchical state machine support
- **Return Values**: Stack-based return value management for interface methods

### Expression Support
- Binary operators with TypeScript semantics (`===`, `!==`)
- Unary operators (`!`, `-`)
- All literal types (strings, numbers, booleans)
- Variable references with automatic `this.` prefixing
- Method call chains
- Template literals for multi-line strings

### Statement Support
- If/else if/else conditionals
- Infinite loops (`while(true)`)
- Return statements with stack management
- Assignment statements
- Action calls
- Continue and break statements

## Usage

### Command Line
```bash
framec -l typescript myspec.frm > output.ts
```

### Attribute Override


## Generated Code Example

```typescript
export class TrafficLight {
    private _compartment: FrameCompartment;
    private _nextCompartment: FrameCompartment | null = null;
    private returnStack: any[] = [];

    constructor() {
        this._compartment = new FrameCompartment('__trafficlight_state_Red');
        this._nextCompartment = null;
        this.returnStack = [null];
        this._frame_kernel(new FrameEvent("$>", null));
    }

    // Interface methods
    public start(): any {
        this.returnStack.push(null);
        const __e = new FrameEvent("start", null);
        this._frame_kernel(__e);
        return this.returnStack.pop();
    }
    
    // ... additional implementation
}
```

## Technical Implementation

### Architecture
- **New Visitor**: `typescript_visitor.rs` implements complete AST traversal
- **CodeBuilder Integration**: Consistent code generation with proper indentation
- **Runtime Classes**: FrameEvent and FrameCompartment for state management
- **Event-Driven**: Message-based communication between states

### Testing
New test suite configuration added:
- `framec_tests/typescript/src/positive_tests/` - Test Frame files
- `framec_tests/runner/configs/typescript_tests.json` - Test configuration
- Generated TypeScript passes strict compilation (`tsc --noEmit`)

## Known Limitations
- Type inference uses `any` type throughout (future enhancement planned)
- No source map generation yet for TypeScript
- Some advanced Frame features not yet implemented:
  - State parameters
  - State stack operations
  - Hierarchical state machines (foundation only)

## Migration Notes
No breaking changes. TypeScript is an additional target language alongside Python and GraphViz.

## Major Achievements

### 1. Set Comprehension Support (NEW)

Frame v0.41 fixes the last remaining collection comprehension gap by implementing full set comprehension support.

#### Problem Solved
- **Issue**: Set comprehensions like `{x*2 for x in range(4)}` were incorrectly parsed as `x in range(4)`
- **Root Cause**: Parser's `dict_or_set_literal` function didn't check for `for` keyword after first expression in set branch

#### Implementation
- **AST Addition**: New `SetComprehensionNode` and `SetComprehensionExprT` variant
- **Parser Fix**: Added comprehension detection in `dict_or_set_literal` function
- **Visitor Support**: Implemented Python code generation for set comprehensions

#### Syntax Examples
```frame
# Basic set comprehension
var doubled = {x * 2 for x in range(5)}         // {0, 2, 4, 6, 8}

# With conditional filtering
var evens = {x for x in range(10) if x % 2 == 0} // {0, 2, 4, 6, 8}

# Complex expressions
var squares = {x * x for x in [1, 2, 3, 4, 5]}  // {1, 4, 9, 16, 25}
```

### 2. String Literal Method Calls

Frame v0.41 introduces support for calling methods directly on string literals, a feature that was missing and causing parse errors in complex string operations.

### Problem Solved

**Issue**: The parser could not handle method calls on string literals like:
```frame
var upper = "hello".upper()                 // Parse error
var stripped = f"{name}".strip()           // Parse error  
var processed = """multiline""".strip()    // Parse error
```

**Root Cause**: The parser's `unary_expression` function only checked for dot operators after identifier expressions, not after literal expressions.

### Implementation Details

#### 1. Parser Enhancement (`parser.rs`)
- **Modified Function**: `unary_expression` at lines 7019-7101
- **New Logic**: Added dot detection after literal expressions
- **Call Chain Building**: Constructs proper call chains starting with literals

```rust
// Check if there's a dot after the literal for method calls like "string".upper()
if self.peek().token_type == TokenType::Dot {
    // Build a call chain starting with the literal...
    call_chain.push_back(CallChainNodeType::CallChainLiteralExprT { 
        call_chain_literal_expr_node: CallChainLiteralExprNode::new(
            literal_expr_node.token_t.clone(),
            literal_expr_node.value.clone(),
        ),
    });
    // Parse method calls...
}
```

#### 2. AST Extension (`ast.rs`)
- **New Variant**: Added `CallChainLiteralExprT` to `CallChainNodeType` enum
- **New Node**: Created `CallChainLiteralExprNode` struct
- **Display Support**: Added formatting for literal expressions in call chains

```rust
pub enum CallChainNodeType {
    // ... existing variants
    CallChainLiteralExprT { 
        call_chain_literal_expr_node: CallChainLiteralExprNode 
    },
}

pub struct CallChainLiteralExprNode {
    pub token_t: TokenType,
    pub value: String,
}
```

#### 3. Visitor Implementation (`python_visitor.rs`)
- **Code Generation**: Added handling for literal expressions in call chains
- **Multiple Locations**: Updated visitor methods to handle the new node type
- **Proper Output**: Generates correct Python syntax for literal method calls

```rust
CallChainNodeType::CallChainLiteralExprT { call_chain_literal_expr_node } => {
    match &call_chain_literal_expr_node.token_t {
        TokenType::String => output.push_str(&format!("\"{}\"", call_chain_literal_expr_node.value)),
        TokenType::FString => output.push_str(&call_chain_literal_expr_node.value),
        // ... other literal types
    }
}
```

#### 4. Bug Fix
- **Issue**: Double RParen consumption in expression list parsing
- **Fix**: Removed redundant `consume(TokenType::RParen)` call since `expr_list()` already consumes it
- **Impact**: Fixed parsing errors in method calls with arguments

### Supported String Literal Types

All Frame string literal types now support method calls:

```frame
// Regular strings
var upper1 = "hello".upper()                    // "HELLO"

// F-strings (formatted)
var upper2 = f"Hello {name}".upper()            // "HELLO FRAME"

// Raw strings  
var lower1 = r"FRAME".lower()                   // "frame"

// Byte strings
var upper3 = b"data".upper()                    // b"DATA"

// Triple-quoted strings
var stripped = """  multiline  """.strip()     // "multiline"

// Prefixed triple-quoted
var lower2 = r"""RAW MULTI""".lower()          // "raw multi"
```

### Grammar Updates

Updated the formal grammar specification:

```bnf
// v0.41: String literal method calls
literal_method_call: string_literal '.' IDENTIFIER '(' arg_list? ')'

primary_expr: IDENTIFIER | NUMBER | string_literal | literal_method_call | SUPERSTRING
            | 'true' | 'false' | 'None'
            | '(' expr ')' | '@'
            | list_literal | dict_literal | set_literal | tuple_literal
            | function_ref
```

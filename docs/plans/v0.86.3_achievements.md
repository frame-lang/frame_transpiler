## 🔧 Critical Fixes Implemented

### 1. **Unary Minus Operator Fix**
**Problem**: `(-2) ** 3` was generating `(!(2) ** 3)` causing syntax errors  
**Solution**: Fixed `OperatorType::Minus|OperatorType::Negated` mapping from `"!"` to `"-"`  
**Location**: `typescript_visitor.rs:1867-1868`  
**Result**: Proper arithmetic negation: `((-2) ** 3)`

### 2. **Matrix Multiplication (@) Operator**
**Problem**: `a @ b` was generating `/* TODO: operator */` comments  
**Solution**: Added `OperatorType::MatMul` handler generating `.matmul()` calls  
**Location**: `typescript_visitor.rs:1889-1897`  
**Result**: `a @ b` → `a.matmul(b)`

### 3. **len() Function Property Access**
**Problem**: `len(numbers)` was generating `(numbers)` instead of `numbers.length`  
**Solution**: Implemented proper `len()` to `.length` conversion with string rewriting  
**Location**: `typescript_visitor.rs:1507-1521`  
**Result**: `len(x)` → `x.length`

### 4. **Numeric Literal .length Bug**
**Problem**: `num < 10` was generating `(num < 10.length)` for numeric literals  
**Solution**: Enhanced Less operator logic to exclude numeric literals from `.length` auto-addition  
**Location**: `typescript_visitor.rs:1801-1803`  
**Result**: Proper numeric comparisons without inappropriate `.length`

### 5. **Enhanced 'in'/'not in' Operators**
**Problem**: Inconsistent handling across object types (arrays, sets, dicts)  
**Solution**: Comprehensive type checking with fallback methods  
**Location**: `typescript_visitor.rs:1838-1888`  
**Result**: Universal compatibility: `x in container` works for all types

## 🛠 Technical Implementation Details

### Systemic .length Property Access Fix:
- **Root Cause**: Overly broad `.length` auto-addition to any alphanumeric string
- **Detection Logic**: Added checks to exclude pure numeric and decimal numbers
- **Scope**: Fixed throughout comparison operators and function calls
- **Validation**: Confirmed working across multiple test categories

### Operator Precedence and Parentheses:
- **Exponentiation**: Special handling for `((-2) ** 3)` syntax
- **Unary Expressions**: Improved parentheses logic to match Python visitor
- **Binary Operations**: Enhanced operator precedence handling

### Function Call Rewriting:
- **Advanced String Manipulation**: Implemented position-aware string truncation and rebuilding
- **Borrow Checker Compliance**: Used `.to_string()` to avoid borrowing conflicts
- **Edge Case Handling**: Robust `.rfind()` with fallback positioning

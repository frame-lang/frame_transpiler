# v0.78 Achievements

## Overview
Version 0.78 improves Python code generation formatting and implements C-style module behavior that prohibits function calls at module scope.

## Key Changes

### 1. Fixed Block Header Formatting
- Resolved extra spacing issues around block header comments in generated Python code
- Ensured consistent single blank line before and after all block headers
- Applied fix to all block types: operations, interface, machine, actions, domain

### 2. C-Style Module Behavior
- Implemented parser validation to prohibit function calls at module scope
- Frame now behaves like C: no statements allowed at module level
- Only declarations (functions, classes, systems, enums, variables) allowed at module scope
- `main()` function is automatically invoked if present (implicit entry point)
- Clear error messages guide users when attempting module-level calls

### 3. Documentation Updates
- Added warnings about deprecated Frame syntax (old `|event|` notation)
- Updated CLAUDE.md with current Frame syntax guidelines
- Clarified that Frame follows C-style module conventions

## Technical Details

### Formatting Fix
The issue was in `python_visitor_v2.rs` where `write_comment()` was automatically adding newlines. Fixed by:
1. Adding `builder.newline()` before writing block header comments
2. Removing automatic newline after comments
3. Ensuring consistent spacing across all block types

### Module-Level Validation
Added validation in `parser.rs` at line 435:
- Detects function calls at module scope
- Provides helpful error message mentioning automatic `main()` invocation
- Allows all declarations but prohibits statements

## Testing
- Created comprehensive formatting tests covering all block types
- Added negative tests for module-level function call restrictions
- Validated with Frame test runner
- Current success rate: 87.4% (325/372 tests passing)

## Known Issues
- System instantiation calls at module level (e.g., `TestSystem()`) not yet caught by validation
- Would require additional parser logic to distinguish from class instantiation

## Migration Guide
For users upgrading to v0.78:
1. Move any module-level function calls into `main()` or other functions
2. Ensure only declarations exist at module scope
3. Add a `main()` function as the entry point if not already present

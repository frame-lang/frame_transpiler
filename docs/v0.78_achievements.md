# Frame v0.78 Series Achievements

## Version v0.78.14 - December 30, 2024
**Test Success Rate: 99.5% (374/376 tests passing)**

### ✅ Major Fix: Cross-System Static Method Calls
Fixed a critical bug where static method calls between systems incorrectly generated `SystemName.self.methodName()` instead of the correct `SystemName.methodName()`.

#### Problem
When calling static methods from another system, the visitor was incorrectly adding a `self.` prefix:
```python
# Incorrect (before fix):
result = UtilitySystem.self.calculate(42)  # AttributeError!

# Correct (after fix):
result = UtilitySystem.calculate(42)
```

#### Solution
Modified `visit_call_chain_expr_node_to_string()` in python_visitor_v2.rs to only add `self.` prefix when the UndeclaredCallT node is the first in the call chain. This correctly distinguishes between:
- Local operation calls (need `self.` prefix)
- Qualified static calls (no `self.` prefix)

#### Impact
- Fixed 2 failing tests: test_static_calls.frm, test_static_comprehensive_v062.frm
- Improved test pass rate from 98.9% to 99.5%
- Static methods now work correctly across all systems

### 📊 Remaining Issues
Only 2 test failures remain, both blocked on a parser bug:
- **Parser Bug**: Functions defined after system definitions cause "Expected '}' found identifier" error
- Affects: test_python_logical_operators.frm, test_state_parameters_simple.frm

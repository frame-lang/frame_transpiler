# Frame v0.58 Achievements - Class Decorators

## Overview
Frame v0.58 introduces Python decorator pass-through support for classes, enabling seamless integration with Python's decorator ecosystem while maintaining Frame's clean syntax.

## Status: ✅ COMPLETE (100% Test Success - 374/374 Tests Passing)

## Key Features

### Class Decorators (NEW in v0.58)
Frame now supports Python decorators on class declarations using the familiar `@decorator` syntax:

```frame
# Simple decorator
@dataclass
class Point {
    var x = 0
    var y = 0
}

# Decorator with arguments
@dataclass(frozen=True)
class ImmutablePoint {
    var x = 0
    var y = 0
}

# Multiple decorators
@dataclass
@total_ordering
class ComparablePoint {
    var x = 0
    var y = 0
}
```

### Implementation Details
- **Pass-through approach**: Decorators are collected as strings and emitted unchanged to Python
- **Class-only support**: Decorators supported on classes, not systems (by design)
- **Method decorators preserved**: `@staticmethod` and `@property` continue to work within classes
- **Parser enhancement**: Smart lookahead to distinguish class decorators from method decorators

### Technical Improvements
- **AST Enhancement**: Added `decorators: Vec<String>` field to ClassNode
- **Parser Logic**: Context-aware decorator parsing that backtracks when not followed by `class`
- **Visitor Update**: Python visitor emits decorators before class definition
- **Backward Compatibility**: All existing class features preserved, including `@staticmethod` and `@property`

## Usage Examples

### Data Classes
```frame
from dataclasses import dataclass

@dataclass
class Person {
    var name = ""
    var age = 0
}

fn main() {
    var p = Person("Alice", 30)
    print(str(p))  # Person(name='Alice', age=30)
}
```

### Property Decorators (Method-level)
```frame
class Temperature {
    fn init(celsius) {
        self._celsius = celsius
    }
    
    @property
    fn fahrenheit() {
        return self._celsius * 9.0 / 5.0 + 32.0
    }
}
```

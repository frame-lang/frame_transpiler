This release includes first-class functions, lambda expressions, complete collection literal support, mathematical operators, and full Python alignment.

## New Features in v0.38

### 1. First-Class Functions
Functions are now first-class values that can be assigned, passed, and returned.

```frame
fn add(a, b) {
    return a + b
}

fn multiply(a, b) {
    return a * b
}

fn get_operation(op_name) {
    if op_name == "add" {
        return add  // Return function reference
    } else {
        return multiply
    }
}

fn apply(func, x, y) {
    return func(x, y)  // Call function parameter
}

fn main() {
    var my_op = get_operation("add")
    var result = apply(my_op, 5, 3)
    print(str(result))  // Output: 8
}
```

### 2. Lambda Expressions
Anonymous functions with Python-style syntax for functional programming.

```frame
fn main() {
    // Simple lambda
    var square = lambda x: x * x
    print(str(square(5)))  // Output: 25
    
    // Multi-parameter lambda
    var add = lambda x, y: x + y
    print(str(add(3, 4)))  // Output: 7
    
    // Lambda with closures
    var multiplier = 10
    var scale = lambda x: x * multiplier
    print(str(scale(5)))  // Output: 50
    
    // Using with higher-order functions
    var numbers = [1, 2, 3, 4, 5]
    var doubled = map(lambda x: x * 2, numbers)
}
```

### 3. Exponent Operator (`**`)
Right-associative power operator for mathematical expressions.

```frame
fn calculate_powers() {
    var base = 2
    var exp = 3
    
    // Basic exponentiation
    var result = base ** exp  // 8
    
    // Right associativity
    var tower = 2 ** 3 ** 2  // 512 (2 ** 9)
    
    // Precedence (higher than multiplication)
    var expr = 2 * 3 ** 2  // 18 (2 * 9)
    
    // Negative exponents
    var fraction = 2 ** -1  // 0.5
    
    return result
}
```

### 4. Empty Set Literal (`{,}`)
Explicit syntax to create empty sets, distinct from empty dictionaries.

```frame
fn collections_demo() {
    // Empty dictionary (unchanged)
    var empty_dict = {}
    
    // Empty set (new syntax)
    var empty_set = {,}
    
    // Non-empty set (unchanged)
    var numbers = {1, 2, 3}
    
    // Operations
    empty_set.add(42)
    var has_42 = 42 in empty_set  // true
    
    return empty_set
}
```

### 5. Python Logical Operators
Complete transition to Python-style boolean operators.

```frame
fn logical_operations(x, y, z) {
    // Boolean AND
    if x and y {
        print("Both true")
    }
    
    // Boolean OR
    if x or y {
        print("At least one true")
    }
    
    // Boolean NOT
    if not z {
        print("z is false")
    }
    
    // Complex expressions
    if (x and y) or (not z) {
        print("Complex condition met")
    }
    
    // Short-circuit evaluation
    var result = x and expensive_function()
    
    return result
}
```

## Complete Collection Support

Frame v0.38 supports all 8 collection literal patterns:

```frame
fn all_collections() {
    // 1. Empty dictionary
    var empty_dict = {}
    
    // 2. Empty set (NEW)
    var empty_set = {,}
    
    // 3. Dictionary literal
    var person = {"name": "Alice", "age": 30}
    
    // 4. Set literal
    var unique = {1, 2, 3, 2, 1}  // {1, 2, 3}
    
    // 5. List literal
    var items = [1, 2, 3, 4, 5]
    
    // 6. Tuple literal
    var coords = (10, 20, 30)
    
    // 7. Dictionary comprehension
    var squares = {x: x**2 for x in range(5)}
    
    // 8. List comprehension
    var evens = [x for x in range(10) if x % 2 == 0]
    
    return squares
}
```

## Technical Details

### Parser Architecture

#### Operator Precedence (Highest to Lowest)
1. **Parentheses**: `()`
2. **Exponent**: `**` (right-associative)
3. **Unary**: `not`, `-` (negation)
4. **Multiplicative**: `*`, `/`, `%`
5. **Additive**: `+`, `-`
6. **Relational**: `<`, `>`, `<=`, `>=`
7. **Equality**: `==`, `!=`
8. **Logical AND**: `and`
9. **Logical OR**: `or`
10. **Logical XOR**: `xor`

#### Two-Pass Parsing
The parser uses a two-pass strategy for function references:
- **Pass 1**: Build symbol table, identify all functions
- **Pass 2**: Resolve references, distinguish function refs from calls

### AST Enhancements


### Python Code Generation

The visitor generates clean, idiomatic Python:


## Migration Guide

### From v0.37 to v0.38

#### Logical Operators
```frame
// Old (no longer supported)
if x && y || !z {
    // ...
}

// New (required)
if x and y or not z {
    // ...
}
```

#### Function References
```frame
// Old (would call function)
var fn = myFunction()

// New (stores reference)
var fn = myFunction
```

#### Empty Sets
```frame
// Old (no way to create empty set)
var s = {}  // This is a dict

// New (explicit empty set)
var s = {,}
```

## Known Issues and Limitations

### Current Limitations
1. **Complex lambda nesting**: Deep lambda nesting may cause parsing issues
2. **Method references**: Can't use `obj.method` as a function reference yet
3. **Lambda statements**: Lambdas limited to single expressions
4. **Type annotations**: No type hints on lambda parameters

### Failed Test Cases (8)
- Complex dictionary patterns
- Advanced lambda compositions
- Unicode character handling in comments
- Future v0.39 features

## Performance Characteristics

- **Compilation**: Slightly faster without C-style operator checks
- **Runtime**: No overhead - generates standard Python
- **Memory**: Minimal AST overhead for new node types
- **Parser**: Two-pass adds ~5% compilation time for function-heavy code

## Best Practices

### Function References
```frame
// Good: Clear function reference
var operation = add

// Good: Higher-order function
fn apply_twice(f, x) {
    return f(f(x))
}

// Avoid: Ambiguous naming
var result = process  // Is this a function or value?
```

### Lambda Usage
```frame
// Good: Simple transformations
var doubled = map(lambda x: x * 2, items)

// Good: Inline filters
var adults = filter(lambda p: p.age >= 18, people)

// Avoid: Complex logic in lambdas
var bad = lambda x: (x * 2 if x > 0 else -x) + 10  // Too complex
```

### Collection Literals
```frame
// Good: Clear intent
var numbers = {1, 2, 3}      // Set
var mapping = {"a": 1}       // Dict
var empty_set = {,}          // Empty set

// Avoid: Ambiguous collections
var unclear = {}  // Dict or set? (It's dict)
```

## Post-Release Fixes (September 2025)

### Lambda in Return Statements
**Fixed**: Return statements now properly parse lambda expressions by using full expression parsing instead of limited equality parsing.

```frame
fn make_adder(n) {
    return lambda x: x + n  // Now works correctly
}
```

### Domain Block Ordering
**Limitation Identified**: Domain blocks with dictionary literals must appear as the last block in system definitions.

```frame
system Example {
    interface:
        test()
    
    machine:
        $Ready {
            test() { return }
        }
    
    // Domain must come last when using dict literals
    domain:
        var settings = {"key": "value"}
}
```

## Known Limitations

1. **Array Indexing with Calls**: Patterns like `array[0](args)` require major parser refactoring
2. **Complex Dictionary Patterns**: Some advanced dictionary operations in conditionals may fail
3. **Domain Block Ordering**: Domain blocks with dictionary literals must appear last in systems
4. **Test Coverage Gaps**: 8 tests remain failing due to edge cases and complex patterns

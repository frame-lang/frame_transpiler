## Major Features

### 1. Matrix Multiplication Operator (@) ✅

Frame now supports Python's matrix multiplication operator (PEP 465), enabling scientific computing with NumPy and similar libraries.

#### Implementation
- **Operator**: `@` for matrix multiplication
- **Compound Assignment**: `@=` for in-place matrix multiplication
- **Precedence**: Same as multiplication and division operators
- **Code Generation**: Direct pass-through to Python's `@` operator

#### Example
```frame
import numpy as np

fn matrix_operations() {
    var a = np.array([[1, 2], [3, 4]])
    var b = np.array([[5, 6], [7, 8]])
    
    # Matrix multiplication
    var result = a @ b  # [[19, 22], [43, 50]]
    
    # In-place operation
    a @= b
    
    # Vector dot product
    var v1 = np.array([1, 2, 3])
    var v2 = np.array([4, 5, 6])
    var dot = v1 @ v2  # 32
}
```

### 2. Bitwise XOR Operator (^) ✅

The bitwise XOR operator, previously reserved for deprecated return syntax, is now available for bitwise operations.

#### Implementation
- **Operator**: `^` for bitwise XOR
- **Compound Assignment**: `^=` for XOR with assignment
- **Precedence**: Between bitwise AND (`&`) and OR (`|`)

#### Example
```frame
fn xor_operations() {
    # Basic XOR
    var a = 5 ^ 3  # Result: 6 (binary: 0101 ^ 0011 = 0110)
    
    # Toggle bits
    var flags = 0b1010
    flags ^= 0b0011  # Result: 0b1001
    
    # XOR encryption pattern
    var data = 42
    var key = 17
    var encrypted = data ^ key
    var decrypted = encrypted ^ key  # Returns to 42
}
```

### 3. Python-Style Comments (#) ✅

Frame has transitioned from C-style comments to Python-style comments, enabling the `//` operator for floor division.

#### Breaking Change
- **Removed**: C-style comments (`//` and `/* */`)
- **Added**: Python-style `#` for single-line comments
- **Retained**: Frame documentation comments `{-- --}` for multi-line

#### Migration
```frame
# Before (v0.39)
// This is a comment
/* Multi-line
   comment */

# After (v0.40)
# This is a comment
# Multi-line comments use
# multiple hash symbols
{-- Or Frame documentation
    comments for blocks --}
```

### 4. Floor Division Operator (//) ✅

With the comment syntax change, Frame can now use `//` for floor division.

```frame
fn division_operations() {
    var regular = 10 / 3   # 3.333...
    var floor = 10 // 3    # 3
    
    # Compound assignment
    var x = 25
    x //= 4  # Result: 6
}
```

### 5. Python Numeric Literals ✅

Support for Python's binary, octal, and hexadecimal numeric notation.

```frame
fn numeric_literals() {
    var binary = 0b1010      # 10 in decimal
    var octal = 0o755        # 493 in decimal
    var hex = 0x1A2B         # 6699 in decimal
    
    # Mix with operators
    var result = 0b1111 ^ 0b1010  # XOR binary numbers
    var perms = 0o644 | 0o111     # OR octal permissions
}
```

## Breaking Changes

### Comment Syntax Migration
**Impact**: All Frame source files must migrate from C-style to Python-style comments.

**Migration Path**:
1. Replace `//` comments with `#`
2. Replace `/* */` blocks with multiple `#` lines or `{-- --}` blocks
3. Update any tooling that parses Frame comments

## Dependencies

### Optional Dependencies
- **NumPy**: Required for matrix multiplication execution (not for transpilation)

1. **Matrix Multiplication**: Requires objects with `__matmul__` method (e.g., NumPy arrays)
2. **Comment Migration**: No automatic migration tool provided
3. **IDE Support**: Some IDEs may need plugin updates for new syntax

# Frame v0.40 Achievements

## Overview
Frame v0.40 represents the completion of Python operator and string literal alignment, bringing Frame to full parity with Python's expressive syntax for operators and string handling.

## Major Features Implemented

### 1. Python String Literals (NEW) ✅
Complete support for Python's rich string literal syntax:

#### F-Strings (Formatted String Literals)
- `f"Hello {name}"` - Embedded expressions
- `f"Sum: {2 + 3}"` - Calculations in strings
- Direct Python f-string generation

#### Raw Strings
- `r"C:\path\to\file"` - No escape processing
- `r"\d{3}-\d{4}"` - Regex patterns
- Preserves all backslashes literally

#### Byte Strings  
- `b"binary data"` - Binary representation
- Direct pass-through to Python

#### Triple-Quoted Strings
- `"""multi-line text"""` - Preserves formatting
- No escaping needed for quotes inside
- Supports prefixes: `r"""raw multi"""`, `f"""formatted multi"""`

#### Percent Formatting
- `"Hello %s" % name` - Classic formatting
- `"%s v%.2f" % (name, version)` - Multiple values
- `"%(key)s" % {"key": value}` - Dictionary formatting

### 2. Python-Style Comments ✅
- **Breaking Change**: Removed C-style `//` and `/* */`
- **New Standard**: `#` for single-line comments
- **Preserved**: `{-- --}` for Frame documentation comments
- **Migration**: All 264 test files successfully converted

### 3. Bitwise XOR Operator ✅
- `^` operator for bitwise XOR
- `^=` compound assignment
- Reclaimed from deprecated return syntax

### 4. Matrix Multiplication (PEP 465) ✅
- `@` operator for matrix multiplication
- `@=` compound assignment
- Compatible with NumPy arrays
- Same precedence as multiplication

### 5. Floor Division ✅
- `//` operator enabled by comment syntax change
- `//=` compound assignment
- Integer division support

### 6. Python Numeric Literals ✅
- Binary: `0b1010`
- Octal: `0o755`
- Hexadecimal: `0x1A2B`
- Original notation preserved in output

## Breaking Changes
1. **Comment Syntax**: Must use `#` instead of `//` for comments
2. **Return Syntax**: Cannot use `^` for return (use `return` keyword)

## Migration Guide

### Comments
```frame

# After (v0.40)

```

## Recent Fixes (2025-09-24)

### 1. Local Variable Scoping Fix
**Problem**: Local variables in for loops were incorrectly treated as state variables, causing KeyError when accessing them.
**Solution**: 
- Removed hardcoded "count" == state variable check
- Added proper tracking of local variables via `current_handler_locals` HashSet
- Fixed in `visit_variable_node_to_string()` method

### 2. Module Variable Initialization Fix
**Problem**: Module variables were being initialized as `None` instead of their actual values (e.g., `PI = None` instead of `PI = 3.14159`)
**Root Causes**:
1. `visit_variable_decl_node` was using `value_rc` instead of `get_initializer_value_rc()`
2. The method wasn't in the AstVisitor trait impl block, so it wasn't being called
3. Module variables were incorrectly tracked as local handler variables

**Solutions**:
- Changed to use `get_initializer_value_rc()` like V1 visitor
- Moved `visit_variable_decl_node` into `impl AstVisitor` block
- Added scope check to not track module variables as locals
- Moved module variable generation to after systems/classes (matching V1 order)

### 3. State Parameter Access Fix
**Problem**: State parameters were not accessible in enter event handlers (NameError)
**Solution**: 
- Fixed scope detection by checking `var_node.scope` instead of `var_node.id_node.scope`
- Now correctly generates `compartment.state_args["param"]` access pattern
- Test `test_state_parameters_simple.frm` now passes

### 4. Async/Sync Interface Method Fix
**Problem**: Non-async interface methods called async kernel without await
**Solution**:
- All interface methods become async when system has async runtime
- Fixed interface method generation to check `system_has_async_runtime` flag
- Now properly awaits kernel calls in all interface methods

### 5. Visitor Trait Organization Fix
**Problem**: Multiple visitor methods were outside the AstVisitor impl block
**Solution**: 
- Moved `visit_function_node` and `visit_variable_decl_node` into AstVisitor impl
- Removed duplicate `visit_state_node` method
- Properly structured impl blocks

### 6. Enum and Action Accessibility Fix
**Problem**: Module-level functions couldn't access domain enums or system actions
**Solution**:
- Generate module-level aliases for domain enums (e.g., `Fruit = FruitSystem_Fruit`)
- Create module-level wrapper functions for system actions using singleton pattern
- Fixed Python name mangling issue in action method calls

### Key Differences V1 vs V2
1. **Code Generation**: V2 uses CodeBuilder with automatic line tracking, V1 uses manual string building
2. **Variable Initialization**: Both now use `get_initializer_value_rc()` 
3. **Module Order**: Both generate module variables after systems/classes
4. **Variable Tracking**: V2 tracks locals in `current_handler_locals`, globals in `global_vars`

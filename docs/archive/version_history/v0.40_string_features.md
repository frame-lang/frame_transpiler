# Frame v0.40 - Python String Features

## Overview
Frame v0.40 adds comprehensive support for Python's advanced string literal features, providing developers with powerful and expressive ways to work with strings.

## New String Features

### 1. F-Strings (Formatted String Literals)
F-strings provide a concise and readable way to embed expressions inside string literals.

```frame
fn example() {
    var name = "Frame"
    var version = 0.40
    
    # Basic f-string
    var msg = f"Hello {name} v{version}!"
    
    # Expressions in f-strings
    var calc = f"Sum: {2 + 3}, Product: {4 * 5}"
    
    print(msg)  # Output: Hello Frame v0.40!
}
```

### 2. Raw Strings
Raw strings treat backslashes as literal characters, making them perfect for file paths and regular expressions.

```frame
fn paths() {
    # Windows paths without escaping
    var path = r"C:\Users\Frame\Documents"
    
    # Regex patterns
    var pattern = r"\d{3}-\d{3}-\d{4}"
    
    # Escape sequences are not processed
    var literal = r"Line 1\nLine 2\tTabbed"
    
    print(path)  # Output: C:\Users\Frame\Documents
}
```

### 3. Byte Strings
Byte strings represent sequences of bytes, useful for binary data handling.

```frame
fn binary() {
    # Basic byte string
    var data = b"Binary data"
    
    # Byte string with hex values
    var hex_data = b"\x48\x65\x6c\x6c\x6f"
    
    print(data)  # Output: b'Binary data'
}
```

### 4. Triple-Quoted Strings
Triple-quoted strings allow multi-line text with preserved formatting.

```frame
fn multiline() {
    # Basic triple-quoted string
    var text = """This is a
    multi-line string
    with preserved formatting"""
    
    # Can contain quotes without escaping
    var quoted = """He said "Hello" and 'Goodbye'"""
    
    print(text)
}
```

### 5. Prefixed Triple-Quoted Strings
Combine prefixes with triple quotes for advanced use cases.

```frame
fn advanced() {
    # Raw triple-quoted string
    var raw_multi = r"""Raw string
    with \n not escaped
    and \t not a tab"""
    
    # F-string triple-quoted
    var name = "Frame"
    var f_multi = f"""Hello {name}
    This is line 2
    This is line 3"""
    
    print(raw_multi)
    print(f_multi)
}
```

### 6. Percent Formatting
Classic Python string formatting with the % operator.

```frame
fn formatting() {
    var name = "Frame"
    var version = 0.40
    var count = 42
    
    # Single value
    var msg1 = "Language: %s" % name
    
    # Multiple values with tuple
    var msg2 = "Name: %s, Version: %.2f, Count: %d" % (name, version, count)
    
    # Dictionary formatting
    var msg3 = "%(lang)s v%(ver).1f" % {"lang": name, "ver": version}
    
    print(msg2)  # Output: Name: Frame, Version: 0.40, Count: 42
}
```

## Migration Guide

### From Regular Strings to F-Strings
```frame
# Before
var msg = "Hello " + name + " version " + str(version)

# After
var msg = f"Hello {name} version {version}"
```

### From Escaped Paths to Raw Strings
```frame
# Before
var path = "C:\\Users\\Frame\\Documents"

# After
var path = r"C:\Users\Frame\Documents"
```

### From Concatenation to Percent Formatting
```frame
# Before
var msg = "Name: " + name + ", Count: " + str(count)

# After
var msg = "Name: %s, Count: %d" % (name, count)
```

## Performance Considerations
- F-strings are generally the fastest string formatting method in Python
- Raw strings have no runtime overhead - they're processed at compile time
- Triple-quoted strings preserve all whitespace, which may impact memory for large texts

## Best Practices

1. **Use f-strings** for most string formatting needs - they're readable and fast
2. **Use raw strings** for file paths and regular expressions
3. **Use triple-quoted strings** for multi-line text like documentation or SQL queries
4. **Use percent formatting** when working with legacy code or specific formatting requirements
5. **Use byte strings** only when dealing with binary data

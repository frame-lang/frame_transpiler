# Frame v0.42 - Python Generator Support with Async Generators

## Overview
Frame v0.42 introduces comprehensive support for Python generator features, including yield expressions, yield from delegation, generator expressions, and async generators. This enhancement enables Frame to support lazy evaluation patterns, memory-efficient iteration, and asynchronous data streaming.

## Key Achievements

### 1. Yield Expressions ✅
- **Basic yield**: `yield value` for yielding single values
- **Bare yield**: `yield` without value (yields None)
- **Proper statement handling**: Yield expressions don't require parentheses when used as statements

### 2. Yield From Delegation ✅
- **Syntax**: `yield from iterable`
- **Use cases**: Delegating to other generators or iterables
- **Example**: `yield from range(10)` or `yield from another_generator()`

### 3. Generator Expressions ✅
- **Basic syntax**: `(expr for var in iterable)`
- **With conditions**: `(expr for var in iterable if condition)`
- **Proper differentiation**: Parser correctly distinguishes generator expressions from tuple literals

### 4. Generator Functions ✅
- **Automatic detection**: Functions containing yield become generators
- **Iterator protocol**: Generated Python functions work as proper generators
- **Memory efficient**: Lazy evaluation for large datasets

### 5. Async Generators ✅
- **Syntax**: `async fn` + `yield` creates async generators
- **Full compatibility**: Leverages v0.35 async support with v0.42 generators
- **Clean output**: No unnecessary parentheses around `await` or `yield`
- **Python async iteration**: Works with Python's `async for` loops

## Implementation Details

### Scanner Enhancements
- Added `Yield` token type for the `yield` keyword
- No separate token for `yield from` (handled as two tokens)

### Parser Enhancements
- New `parse_yield_expression()` method for yield parsing
- Enhanced `expr_list_or_tuple()` to detect generator expressions
- Lookahead logic to differentiate generators from tuples

### AST Additions
```rust
// New expression types
YieldExprT { yield_expr_node: YieldExprNode }
YieldFromExprT { yield_from_expr_node: YieldFromExprNode }
GeneratorExprT { generator_expr_node: GeneratorExprNode }

// Node structures
pub struct YieldExprNode {
    pub expr: Option<Box<ExprType>>,  // Optional for bare yield
}

pub struct YieldFromExprNode {
    pub expr: Box<ExprType>,  // Required iterable
}

pub struct GeneratorExprNode {
    pub expr: Box<ExprType>,
    pub target: String,
    pub iter: Box<ExprType>,
    pub condition: Option<Box<ExprType>>,
}
```

### Python Visitor Implementation
- Generates proper Python yield statements
- No unnecessary parentheses for standalone yields, await, or yield from
- Correct generator expression syntax with conditions
- Async generator support through `async def` with `yield`

## Examples

### Basic Generator Function
```frame
fn count_up_to(n) {
    var i = 0
    while i < n {
        yield i
        i = i + 1
    }
}

# Usage
for val in count_up_to(5) {
    print("Value: " + str(val))
}
```

### Yield From Delegation
```frame
fn delegated_generator() {
    yield from range(3)
    yield from [10, 20, 30]
}
```

### Generator Expressions
```frame
# Basic generator expression
var squares = (x * x for x in range(10))

# With condition
var evens = (x for x in range(100) if x % 2 == 0)

# Complex expressions
var processed = (str(x).upper() for x in items if x > 0)
```

### Async Generators (NEW!)
```frame
import asyncio

# Async generator function
async fn async_data_stream(n) {
    var i = 0
    while i < n {
        await asyncio.sleep(0.1)  # Simulate async I/O
        yield f"Data chunk {i}"
        i = i + 1
    }
}

# Usage with Python's async for (in Python code)
# async for chunk in async_data_stream(10):
#     print(chunk)
```

## Known Limitations
1. **Bare yield parsing**: When a bare `yield` is followed by a valid expression on the next line, the parser may interpret it as `yield expression`. This is a parsing ambiguity that exists in Python as well.

2. **Module-level iteration**: For loops at module level (outside functions) have pre-existing issues unrelated to generators.

## Technical Notes
- Generator expressions properly handle the "for" keyword lookahead
- Yield expressions are treated as valid expression statements
- The visitor correctly avoids wrapping standalone yields in parentheses

## Migration Guide
Frame v0.42 generators map directly to Python generators:
- Frame `yield x` → Python `yield x`
- Frame `yield from iter` → Python `yield from iter`
- Frame `(x for x in y)` → Python `(x for x in y)`

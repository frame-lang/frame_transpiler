# Frame v0.52 Achievements - Multiple Assignment and Tuple Unpacking

## Overview
Frame v0.52 introduces Python-style multiple assignment and tuple unpacking capabilities, enabling more elegant and Pythonic code patterns in Frame programs.

## Key Features

### 1. Multiple Assignment ✅
Assign values to multiple variables in a single statement:
```frame
# Basic multiple assignment
x, y = 10, 20

# Variable swapping
a, b = b, a
```

### 2. Tuple Unpacking ✅
Unpack values from tuples into individual variables:
```frame
var t = (100, 200, 300)
var p, q, r = 0, 0, 0
p, q, r = t  # Unpacks tuple into three variables
```

### 3. Function Return Unpacking ✅
Unpack multiple return values from functions:
```frame
fn get_coordinates() {
    return (42, 73)
}

var lat, lon = 0, 0
lat, lon = get_coordinates()
```

### 4. Expression Assignment ✅
Multiple assignment works with complex expressions:
```frame
n1, n2, n3 = n1 + 1, n2 * 2, n3 ** 2
```

## Implementation Details

### Parser Changes
- Enhanced `assignment_or_lambda()` to detect comma-separated assignment targets
- Added RHS tuple detection for comma-separated values
- Proper handling of tuple literals vs multiple assignment distinction

### AST Enhancements
- Added `is_multiple_assignment` flag to `AssignmentExprNode`
- Added `l_values` vector for multiple assignment targets
- New constructor `new_multiple()` for creating multiple assignment nodes

### Code Generation
- Python visitor generates proper comma-separated targets
- Automatic tuple wrapping for RHS when multiple values present
- Preserves Python's native unpacking semantics

## Known Limitations

### List Literal Issue
When using list literals with comma-separated values, Frame currently converts them to a tuple within a list:
```frame
# This:
var lst = [1, 2, 3]
# Becomes:
lst = [(1, 2, 3)]  # List with one tuple instead of three elements
```

**Workaround**: Build lists programmatically:
```frame
var lst = [1]
lst.append(2)
lst.append(3)
```

### Variable Declaration
Multiple variable declarations (`var x, y = 10, 20`) only register the first variable in the symbol table. Use separate declarations then assignment:
```frame
var x = 0
var y = 0
x, y = 10, 20
```

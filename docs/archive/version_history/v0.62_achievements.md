# Frame v0.62 Achievements - Semantic Call Resolution

**Release Date**: 2025-09-21  
**Branch**: v0.30  
**Test Success Rate**: 100% (379/379 tests passing)

## Overview

Frame v0.62 introduces **semantic call resolution during parsing**, a major architectural improvement that moves complex call chain analysis from the visitor layer to the parser where it semantically belongs. This foundational change simplifies code generation, improves maintainability, and sets the stage for advanced type checking and IDE support.

## Major Achievement: Parser-Based Semantic Analysis

### The Problem

Previously, the Python visitor contained 350+ lines of complex logic to determine what type of call each expression represented (action, operation, static method, etc.). This logic was duplicated across visitors and made the codebase hard to maintain.

### The Solution

v0.62 introduces a clean two-pass semantic analysis architecture:

1. **First Pass**: Builds the symbol table (existing behavior)
2. **Second Pass**: Performs semantic analysis with the new `SemanticAnalyzer` module

## Key Components

### 1. ResolvedCallType Enum

A comprehensive enum that categorizes all possible call types in Frame:

```rust
pub enum ResolvedCallType {
    Action(String),           // Internal action methods
    Operation(String),        // System operations
    SystemOperation {         // Static system operations
        system: String,
        operation: String,
        is_static: bool,
    },
    ClassMethod {            // Class methods (static/instance)
        class: String,
        method: String,
        is_static: bool,
    },
    ModuleFunction {         // Module-scoped functions
        module: String,
        function: String,
    },
    External(String),        // External functions/builtins
}
```

### 2. SemanticAnalyzer Module

A dedicated semantic analysis module that:
- Resolves call expressions to their semantic type
- Tracks context (system, class, function scope)
- Leverages the symbol table built in the first pass
- Stores resolution results in AST nodes

### 3. Enhanced AST Nodes

`CallExprNode` enhanced with:
- `context`: CallContextType for initial classification
- `resolved_type`: Option<ResolvedCallType> for semantic resolution

## Implementation Details

### Parser Integration

The parser now calls `resolve_call_expr()` at all 5 CallExprNode creation points:
1. Method calls on literals (e.g., `"hello".upper()`)
2. Indexed calls (e.g., `funcs[0](args)`)
3. Standard function calls
4. Self method calls (e.g., `self.method()`)
5. Chained property calls (e.g., `self.prop.method()`)

### Feature Flag Control

Gradual migration enabled via environment variable:
```bash
FRAME_SEMANTIC_RESOLUTION=1 framec -l python_3 file.frm
```

### Debug Support

Full debug output showing resolution results:
```bash
FRAME_TRANSPILER_DEBUG=1 FRAME_SEMANTIC_RESOLUTION=1 framec -l python_3 file.frm
```

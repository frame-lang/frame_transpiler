# v0.78.1 Achievements

## Overview
Version 0.78.1 is a bug fix release that resolves Bug #10, where static method calls were incorrectly generated with an extra "self" prefix in the call chain.

## Key Changes

### Fixed Bug #10: Static Method Call Generation
- **Issue**: When calling a static method from within a call chain (e.g., `TestService.getDefaultConfig()`), the transpiler incorrectly generated `TestService.self.getDefaultConfig()`
- **Root Cause**: The `UndeclaredCallT` node handler was calling `visit_call_expression_node_to_string()` which added "self." prefix for operations
- **Solution**: Modified the handler to directly output the method name and arguments when part of a call chain, avoiding the incorrect "self." prefix
- **Impact**: Static method calls now work correctly in all contexts

## Technical Details

### Code Change
Modified `python_visitor_v2.rs` at line 3536-3549 in the `visit_call_chain_expr_node_to_string` method:
- When handling `UndeclaredCallT` nodes that are part of a call chain
- Instead of calling `visit_call_expression_node_to_string()` which adds "self." for operations
- Now directly outputs the method name and arguments
- This prevents incorrect "self." prefix for static method calls like `TestService.getDefaultConfig()`

# Frame v0.46 Achievements - Class Support

**Release Date**: 2025-09-11  
**Branch**: v0.30  
**Test Success Rate**: 100% (327/327 tests passing)

## Executive Summary

Frame v0.46 introduces comprehensive object-oriented programming support through classes, enabling developers to use familiar OOP patterns alongside Frame's state machine paradigm. This release brings inheritance, properties, decorators, and special methods to the Frame language.

## Major Features Implemented

### 1. Class Declarations
- **Basic Classes**: Simple class definitions with methods and variables
- **Class Variables**: Shared variables at class level
- **Instance Variables**: Created via `self.varname = value`
- **Constructor Methods**: Methods named `init` or `__init__` become constructors
- **Implicit Self**: Method signatures don't include `self` (added automatically)

### 2. Class Inheritance
- **Single Inheritance**: `extends` keyword for inheriting from parent classes
- **Method Overriding**: Child classes can override parent methods
- **Super Calls**: `super().__init__()` syntax for calling parent methods
- **Inheritance Chain**: Proper method resolution order

### 3. Method Decorators
- **@staticmethod**: Static methods without self/cls parameters
- **@classmethod**: Class methods receiving `cls` parameter
- **@property**: Property decorators for getters
- **Property Setters**: `@property_name.setter` decorators
- **Property Deleters**: `@property_name.deleter` decorators

### 4. Special Methods
- **Dunder Methods**: Full support for Python special methods
- **Operator Overloading**: `__add__`, `__eq__`, `__len__`, etc.
- **String Representation**: `__str__` and `__repr__` methods
- **Initialization**: `__init__` as alternative constructor name
- **Passthrough Support**: All Python special methods work without validation

## Code Examples

### Basic Class with Inheritance
```frame
class Animal {
    var species = "Unknown"
    var count = 0
    
    fn init(name) {
        self.name = name
        Animal.count = Animal.count + 1
    }
    
    fn speak() {
        return "Some sound from " + self.name
    }
    
    @classmethod
    fn get_count(cls) {
        return cls.count
    }
    
    @staticmethod
    fn info() {
        return "Animals are living organisms"
    }
}

class Dog extends Animal {
    fn init(name, breed) {
        super().__init__(name)
        self.breed = breed
        self.species = "Canis familiaris"
    }
    
    fn speak() {
        return "Woof! My name is " + self.name
    }
    
    fn fetch() {
        return self.name + " is fetching!"
    }
}
```

### Properties Example
```frame
class Temperature {
    fn init(celsius) {
        self._celsius = celsius
    }
    
    @property
    fn celsius() {
        return self._celsius
    }
    
    @celsius.setter
    fn celsius(value) {
        if value < -273.15 {
            print("Error: Temperature below absolute zero")
            return
        }
        self._celsius = value
    }
    
    @property
    fn fahrenheit() {
        return self._celsius * 9.0 / 5.0 + 32.0
    }
    
    @fahrenheit.setter
    fn fahrenheit(value) {
        self.celsius = (value - 32.0) * 5.0 / 9.0
    }
}
```

### Special Methods Example
```frame
class Vector {
    fn __init__(x, y) {
        self.x = x
        self.y = y
    }
    
    fn __str__() {
        return "Vector(" + str(self.x) + ", " + str(self.y) + ")"
    }
    
    fn __add__(other) {
        return Vector(self.x + other.x, self.y + other.y)
    }
    
    fn __eq__(other) {
        return self.x == other.x and self.y == other.y
    }
    
    fn __len__() {
        return int((self.x * self.x + self.y * self.y) ** 0.5)
    }
}
```

## Migration Guide

### From Functions to Classes
```frame
# Before (v0.45)
fn create_point(x, y) {
    return {"x": x, "y": y}
}

fn distance(p1, p2) {
    var dx = p1["x"] - p2["x"]
    var dy = p1["y"] - p2["y"]
    return ((dx * dx) + (dy * dy)) ** 0.5
}

# After (v0.46)
class Point {
    fn init(x, y) {
        self.x = x
        self.y = y
    }
    
    fn distance_to(other) {
        var dx = self.x - other.x
        var dy = self.y - other.y
        return ((dx * dx) + (dy * dy)) ** 0.5
    }
}
```

## Known Limitations

1. **Single Inheritance Only**: Multiple inheritance not supported
2. **No Access Modifiers**: All class members are public
3. **No Abstract Classes**: Interface-style contracts not enforced
4. **No Metaclasses**: Python metaclass features not exposed

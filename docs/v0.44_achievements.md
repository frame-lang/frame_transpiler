# Frame v0.44 - Pattern Matching with match-case

## Overview
Frame v0.44 introduces Python-style pattern matching using the `match` and `case` keywords, bringing powerful structural pattern matching capabilities to Frame. This feature enables cleaner, more expressive code for complex conditional logic.

## Key Features

### 1. Match Statement Syntax ✅
```frame
match expression {
    case pattern1 {
        # statements
    }
    case pattern2 if guard_condition {
        # statements with guard
    }
    case _ {
        # default case
    }
}
```

### 2. Supported Pattern Types

#### Literal Patterns ✅
Match against specific values:
```frame
match value {
    case 42 {
        return "the answer"
    }
    case "hello" {
        return "greeting"
    }
    case true {
        return "boolean true"
    }
    case None {
        return "null value"
    }
}
```

#### Capture Patterns ✅
Bind matched values to variables:
```frame
match value {
    case 0 {
        return "zero"
    }
    case x {
        return "captured: " + str(x)
    }
}
```

#### Wildcard Pattern ✅
Match anything without binding:
```frame
match value {
    case [_, _, 3] {
        # Matches any list with 3 as third element
        return "ends with 3"
    }
    case _ {
        # Matches anything
        return "default"
    }
}
```

#### Sequence Patterns ✅
Match lists and tuples:
```frame
match lst {
    case [] {
        return "empty"
    }
    case [x] {
        return "single: " + str(x)
    }
    case [x, y] {
        return "pair: " + str(x) + ", " + str(y)
    }
    case [x, y, z] {
        return "triple"
    }
}
```

#### Mapping Patterns ✅
Match dictionary structures:
```frame
match dct {
    case {"type": "error"} {
        return "error type"
    }
    case {"type": t, "code": c} {
        return "type: " + t + " code: " + str(c)
    }
    case {"name": n, "age": a} {
        return "person: " + n
    }
}
```

#### Guard Clauses ✅
Add conditions to patterns:
```frame
match value {
    case x if x < 0 {
        return "negative"
    }
    case x if x == 0 {
        return "zero"
    }
    case x if x > 0 and x < 10 {
        return "small positive"
    }
}
```

#### AS Patterns ✅
Bind entire matched patterns to variables:
```frame
match value {
    case [x, y] as lst {
        # lst contains the whole list, x and y contain elements
        return "list: " + str(lst)
    }
    case x as val {
        # val contains the same as x
        return "value: " + str(val)
    }
}
```

### 4. OR Patterns ✅
Match multiple alternatives using `or` keyword:
```frame
match value {
    case 1 or 2 or 3 {
        return "small number"
    }
    case "yes" or "y" or "true" {
        return "affirmative"
    }
}
```

### 5. Star Patterns ✅
Capture remaining elements in sequences:
```frame
match lst {
    case [first, *rest] {
        # first gets first element, rest gets remaining list
        return "first: " + str(first) + ", rest: " + str(rest)
    }
    case [first, *middle, last] {
        # first gets first, last gets last, middle gets everything between
        return "edges and middle"
    }
}
```

### 6. Nested Patterns ✅
Match complex nested structures:
```frame
match data {
    case [1, [2, 3]] {
        return "specific nested list"
    }
    case [x, [y, z]] {
        return "nested with captures"
    }
    case {"data": [x, y]} {
        return "dict containing list"
    }
}
```

## Implementation Details

### Scanner Changes
- Added `Match` and `Case` keywords to `TokenType` enum
- Keywords registered in scanner's keyword map

### AST Additions
- `MatchStmtNode`: Represents match statement with expression and cases
- `CaseNode`: Individual case with pattern, optional guard, and statements
- `PatternNode`: Enum representing different pattern types
  - `Literal`: Literal value patterns
  - `Capture`: Variable binding patterns
  - `Wildcard`: Underscore pattern
  - `Sequence`: List/tuple patterns
  - `Mapping`: Dictionary patterns
  - `Class`: Class-based patterns (limited support)
  - `Or`: OR patterns for multiple alternatives
  - `As`: AS patterns for binding matched values
  - `Star`: Star patterns for sequence unpacking

### Parser Implementation
- `match_statement()`: Parses match statements
- `parse_pattern()`: Recursive pattern parser supporting all pattern types including AS patterns
- `parse_pattern_inner()`: Helper for inner pattern parsing before AS clause
- `parse_literal()`: Helper for literal patterns
- Guard clause parsing with `if` expressions

### Python Visitor
- `visit_match_stmt_node()`: Generates Python `match` statement
- `visit_case_node()`: Generates `case` clauses with guards
- `visit_pattern_node()`: Pattern-specific code generation
- Proper indentation and statement handling

## Limitations

### Not Supported Yet
1. **Class Patterns**: Limited support, uses tuple workaround (need Frame classes first)

### Known Issues
- Dictionary pattern keys must be string literals or identifiers
- Class patterns are simulated using tuples

## Migration Guide

### From if-elif-else Chains
Before (v0.43):
```frame
fn process(value) {
    if value == 1 {
        return "one"
    } elif value == 2 {
        return "two"
    } elif value == 3 {
        return "three"
    } else {
        return "other"
    }
}
```

After (v0.44):
```frame
fn process(value) {
    match value {
        case 1 {
            return "one"
        }
        case 2 {
            return "two"
        }
        case 3 {
            return "three"
        }
        case _ {
            return "other"
        }
    }
}
```

### Structural Matching
```frame
fn handle_response(response) {
    match response {
        case {"status": 200, "data": data} {
            return process_success(data)
        }
        case {"status": 404} {
            return handle_not_found()
        }
        case {"status": code} if code >= 500 {
            return handle_server_error(code)
        }
        case _ {
            return handle_unknown()
        }
    }
}
```

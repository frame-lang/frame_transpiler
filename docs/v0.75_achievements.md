# Frame v0.75 Achievements - CodeBuilder Architecture

**Released**: December 22, 2025  
**Major Achievement**: Complete replacement of fragile manual line tracking with robust CodeBuilder architecture

## Executive Summary

Frame v0.75 represents a major architectural improvement in code generation and source mapping. The new CodeBuilder module provides automatic character-level tracking, eliminating all manual offset adjustments and creating a solid foundation for perfect source mappings.

## Key Achievements

### 1. CodeBuilder Module (✅ Complete)
- **Automatic Position Tracking**: Every character written is automatically tracked
- **Line/Column Management**: Automatic handling of newlines and position updates
- **Deferred Mapping**: `map_next()` method for associating mappings with next write
- **Immediate Mapping**: `add_source_mapping()` for direct mapping associations
- **Child Builders**: Support for complex nested code generation patterns
- **Fragment Composition**: Build code non-linearly and merge results

### 2. PythonVisitorV2 (✅ Complete)
- **Full Frame Support**: All language features implemented
- **CodeBuilder Integration**: Uses CodeBuilder for all output
- **Clean Architecture**: No manual line tracking or offset adjustments
- **Backward Compatible**: Original visitor available via `USE_PYTHON_V1`
- **Default Visitor**: V2 is now the default for Python generation

### 3. Source Mapping Improvements
- **Character Precision**: Every character tracked, not just lines
- **Automatic Handling**: No manual offsets or adjustments needed
- **Robust Architecture**: Changes to generation don't break mappings
- **Future-Proof**: Easy to add/remove generated code without breaking

## Technical Implementation

### CodeBuilder API
```rust
pub struct CodeBuilder {
    output: String,
    current_position: Position,
    mappings: Vec<SourceMapping>,
    pending_mapping: Option<usize>,
}

impl CodeBuilder {
    // Write text and track position automatically
    pub fn write(&mut self, text: &str)
    
    // Set mapping for next write operation
    pub fn map_next(&mut self, frame_line: usize)
    
    // Add immediate mapping
    pub fn add_source_mapping(&mut self, frame_line: usize)
    
    // Create child builder for nested generation
    pub fn child(&self) -> CodeBuilder
    
    // Merge child builder results
    pub fn merge(&mut self, child: CodeBuilder)
}
```

### Usage in PythonVisitorV2
```rust
fn visit_function(&mut self, node: &FunctionNode) {
    // Automatic mapping - next write gets Frame line association
    self.builder.map_next(node.line);
    
    // Write function definition - position tracked automatically
    self.builder.write("def ");
    self.builder.write(&node.name);
    self.builder.write("(");
    
    // Parameters
    for (i, param) in node.params.iter().enumerate() {
        if i > 0 {
            self.builder.write(", ");
        }
        self.builder.write(param);
    }
    
    self.builder.write("):\n");
    // Line tracking automatic - no manual incrementing!
}
```

## Impact on Development

### Before (Manual Tracking)
```rust
// Fragile manual tracking
self.current_line += 1;
self.add_code("def function():\n");
self.current_line += 1; // Easy to forget!
// Complex offset calculations
let offset = if has_comment { 2 } else { 1 };
self.add_mapping(node.line, self.current_line + offset);
```

### After (CodeBuilder)
```rust
// Automatic tracking
self.builder.map_next(node.line);
self.builder.write("def function():\n");

```

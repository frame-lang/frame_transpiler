# v0.86.20 — TypeScript Runtime Parity Upgrades

- Implemented Python-style string helpers in the TS runtime (`FrameString`): search (`find`/`rfind`/`index`/`rindex`), transforms (`upper`/`lower`/`title`/`capitalize`/`swapcase`), splitting/partitioning variants, padding/formatting, encoding, and character-class checks. Prototype patching now allows generated JS to call the Python method names directly.
- Extended `FrameCollections`/`FrameDict` utilities so list-counting spans strings, `defaultdict` factories emit proper functions, `Counter` arithmetic works, and dict instances expose `.keys()/.values()/.items()` for constructor tests.
- Visitor updates detect string/list/dict calls in chains and emit the new runtime helpers, while list/dict comprehensions stay routed through `FrameDict`/`FrameCollections`.
- Result: `python3 framec_tests/runner/frame_test_runner.py --languages typescript --categories data_types --framec ./target/release/framec` now completes with 66/66 execution success.

## 🐛 Bugs Fixed

### Bug #54-56: TypeScript External API Method Call Issue
**Problem:** When Frame actions had names matching external API methods (e.g., `spawn`, `createServer`), the TypeScript visitor incorrectly generated `this._action_` prefixes for external API calls.

**Example of the issue:**
```frame
actions:
    spawn(command, args, options) {
        return child_process.spawn(command, args, options)
    }
```

**Previously generated (incorrect):**
```typescript
return child_process.this._action_spawn(command, args, options);  // ❌
```

**Now generates (correct):**
```typescript
return child_process.spawn(command, args, options);  // ✅
```

**Solution:** Modified the TypeScript visitor to check context when processing method calls:
- If a method call is the first in a call chain and matches an action name → use `this._action_` prefix
- If a method call appears after a module/object reference → treat as external API call without prefix

## 🔧 Technical Details

### Files Modified
- `framec/src/frame_c/visitors/typescript_visitor.rs` (lines 1368-1382)
  - Modified `visit_call_expr_node_to_string_with_context` to check `is_first_in_chain` parameter
  - Only adds `_action_` or `_operation_` prefix when the call is first in the chain

### Key Insight
The bug occurred because the visitor was checking if a method name existed in the action_names set without considering the context. External API methods that happened to share names with Frame actions were incorrectly being prefixed.

# v0.86.19 — TypeScript Dictionary Runtime Stabilization

- Promoted `FrameDict` to first-class runtime facade for TypeScript, normalizing complex keys, constructor inputs, unions, and comprehension output.
- Routed `dict()`, `dict.fromkeys()`, comprehensions, and helper calls (`get`, `setdefault`, `update`) through the new runtime helpers so transpiled code now matches Python semantics.
- Added JSON and literal-eval shims (`FrameRuntime.jsonLoads`, `jsonDumps`, `literalEval`) to unblock dynamic configuration parsing in the TypeScript target.
- Tightened comprehension emission by wrapping tuple destructuring and leveraging `FrameRuntime.iterable`, restoring deterministic code generation for `data_types` suites.
- Switched first-use assignments to `var` so counter-style patterns and chained lookups emit valid ES5 without leaking undeclared identifiers.

**Verification**

- `cargo build --release`
- `python3 framec_tests/runner/frame_test_runner.py --languages typescript --categories data_types --transpile-only --framec ./target/release/framec`

# Frame v0.66 Achievements - Explicit Self/System Call Syntax

**Release Date**: December 18, 2024  
**Version**: v0.66  
**Branch**: v0.30  
**Status**: ✅ Complete - 100% Test Success (379/379 tests passing)

## Overview

Frame v0.66 establishes explicit `self.` prefix as a requirement for all internal method calls within systems, aligning with Python semantics and improving code clarity. Additionally, semantic call resolution is now an integral part of the parser, removing the previous feature flag.

## Major Achievements

### 1. Explicit Self/System Call Syntax ✅

All internal method calls within systems now require explicit prefixes:

```frame
system TrafficLight {
    operations:
        log(msg) {
            print(msg)
        }
        
    interface:
        next()
        
    machine:
        $Red {
            auto_cycle() {
                // v0.66: All calls require explicit self prefix
                for i in range(3) {
                    self.next()        // Interface method call
                    self.log("Step")   // Operation call
                    self.wait()        // Action call
                }
            }
        }
        
    actions:
        wait() {
            print("Waiting...")
        }
}
```

### 2. SystemInterface Resolution Type ✅

Added new `ResolvedCallType::SystemInterface` variant to handle interface methods called from within the system:

```rust
pub enum ResolvedCallType {
    Action(String),              // Internal action call
    Operation(String),           // Internal operation call  
    SystemInterface {            // Interface method call from within system (NEW)
        system: String,
        method: String,
    },
    SystemOperation { ... },     // Qualified system operation
    ClassMethod { ... },         // Class method
    ModuleFunction { ... },      // Module function
    External(String),            // External function
}
```

### 3. Semantic Resolution Always Enabled ✅

Removed the `FRAME_SEMANTIC_RESOLUTION` environment variable and feature flag:
- Semantic resolution is now always active
- Two-pass parsing is the standard behavior
- All calls are resolved to their semantic types during parsing

### 4. Enhanced Semantic Analyzer ✅

The `SemanticCallAnalyzer` now properly resolves:
- Action calls (with or without underscore prefix)
- Operation calls
- Interface method calls from within systems
- Static operation calls
- Class method calls
- Module function calls
- External/built-in function calls

## Technical Implementation

### Parser Changes

1. **Removed Feature Flag**:
   - Deleted `enable_semantic_resolution` field from Parser struct
   - Removed environment variable checks
   - Semantic resolution is now always performed in second pass

2. **Enhanced Call Resolution**:
   - Added `is_interface_method_in_system()` to check interface methods
   - Updated `resolve_self_call()` to handle interface methods
   - SystemInterface calls generate proper `self.method()` code

### Visitor Changes

Python visitor properly handles all resolved call types:
- Actions: Adds underscore prefix and `self.`
- Operations: Adds `self.` prefix
- SystemInterface: Generates `self.method()` for interface calls
- External: No prefix needed

## Migration Guide

### Before v0.66
```frame
system Example {
    machine:
        $Start {
            process() {
                _doAction()      // Implicit action call
                calculate()      // Implicit operation call  
                next()          // Implicit interface call
            }
        }
}
```

### After v0.66
```frame
system Example {
    machine:
        $Start {
            process() {
                self.doAction()      // Explicit action call
                self.calculate()     // Explicit operation call
                self.next()         // Explicit interface call
            }
        }
}
```

## Benefits

1. **Code Clarity**: Explicit prefixes make it clear when calling internal methods
2. **Python Alignment**: Matches Python's requirement for `self.` prefix
3. **Consistency**: All internal calls follow the same pattern
4. **Better Tooling**: Easier for IDEs and tools to analyze Frame code
5. **Reduced Ambiguity**: Clear distinction between internal and external calls

## Breaking Changes

⚠️ **All Frame code must be updated to use explicit `self.` prefix for internal calls**

This is a breaking change that requires updating existing Frame source code. The transpiler will no longer accept implicit action/operation/interface calls.

# Frame v0.31 Achievements

## Release Date: 2025-08-31

## Major Features Implemented

### 1. Native Import Statement Support
- **Simple imports**: `import math`, `import json`
- **Aliased imports**: `import numpy as np`, `import os.path as osp`
- **From imports**: `from collections import defaultdict, OrderedDict`
- **Wildcard imports**: `from typing import *`
- **Dotted module names**: Full support for packages like `os.path`
- **Python-first design**: Primary target is Python, with extensibility for other languages

### 2. Self Expression Enhancement
- **Standalone self**: Can now use `self` as a complete expression (e.g., `jsonpickle.encode(self)`)
- **No backticks required**: Direct usage without escape syntax
- **Maintains compatibility**: `self.variable` and `self.method()` continue to work

### 3. Static Method Validation
- **Parse-time validation**: Static operations marked with `@staticmethod` cannot use `self`
- **Clear error messages**: "Cannot use 'self' in a static operation (marked with @staticmethod)"
- **Proper scoping**: Operations are only static when explicitly marked with `@staticmethod` attribute

### 4. Operations Behavior Correction
- **Instance methods by default**: Operations without `@staticmethod` are instance methods with implicit `self` parameter
- **Static only when declared**: Operations must be explicitly marked with `@staticmethod` to be static
- **Consistent with Python**: Aligns Frame's behavior with Python's method conventions

## Implementation Details

### Scanner Changes
- Added `Import`, `From`, `As` token types
- Added keywords to scanner map for import support

### Parser Changes
- Added `parse_import_statement()` and `parse_from_import_statement()` methods
- Support for dotted module names in imports
- Added validation for static operations using `self`
- Modified `parse_self_context()` to allow standalone self expressions

### AST Changes
- Added `ImportNode` and `ImportType` structures
- Added `Import` variant to `ModuleElement` enum
- Support for four import types: Simple, Aliased, FromImport, FromImportAll

### Python Visitor Changes
- Added `visit_import_node()` implementation
- Updated `run_v2()` to process import statements
- Fixed operations to only be static when attributed
- Added self parameter handling for non-static operations

## Native Python Integration

Frame v0.31 now provides seamless access to Python's built-in functions without requiring Frame-specific implementations. The `print` function, `str`, `int`, `len`, and other Python built-ins are directly accessible, making Frame code more Pythonic and reducing the transpiler complexity.

## Summary
Frame v0.31 represents a significant step forward in language usability, particularly for Python developers. The addition of native import statements eliminates the need for backticks in most common scenarios, while the self expression enhancements and static method validation provide a more intuitive and safer development experience. The move to native Python function access further simplifies the language and improves interoperability.

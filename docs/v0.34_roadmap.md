# Frame v0.34 Roadmap - Module System & Multi-Target Support

**Target Release**: Q1 2024  
**Branch**: v0.34  
**Focus**: Module system implementation and Rust as first-class target language

## Executive Summary

Frame v0.34 will introduce a comprehensive module system inspired by Python but enhanced with explicit nested modules. The FSL will become an optional import rather than default, preventing namespace pollution and conflicts. Additionally, Rust will be added as a first-class compilation target.

## Core Features

### 1. Module System Implementation

#### File-as-Module Architecture
- Each `.frm` file automatically creates a module namespace
- Module name derived from filename (like Python)
- No explicit module declaration needed at file level
- Clean, predictable project structure

#### Nested Module Support
```frame
// File: utils.frm (creates 'utils' module)

fn helper() { }  // utils.helper

// Explicit nested module
module string {
    fn format(s: str): str { }  // utils.string.format
}

module math {
    fn calculate(x: int): int { }  // utils.math.calculate
    
    module advanced {
        fn integral(f: fn): float { }  // utils.math.advanced.integral
    }
}
```

#### Import System
```frame
// Various import styles
import utils                        // Import entire module
import utils.{helper}              // Import specific function
import utils.string.format         // Import nested function
from utils.math import calculate   // Python-style syntax
import utils.math.{calculate, advanced}  // Multiple imports
```

### 2. FSL as Optional Import

#### No Default FSL
```frame
// FSL operations NOT available by default
fn main() {
    var s = str(42)  // ERROR: 'str' is not defined
}
```

#### Explicit FSL Import
```frame
import fsl.{str, int, list, map}  // Must explicitly import FSL

fn main() {
    var s = str(42)        // Now works - FSL's str
    var nums = list()      // FSL's list constructor
}
```

#### User-Defined Replacements
```frame
// Without FSL import, users can define their own versions
fn str(x: any): string {
    return custom_stringify(x)
}

fn list(): array {
    return custom_array_new()
}
```

### 3. Symbol Resolution & Conflict Prevention

#### Strict No-Conflict Rules
```frame
import fsl.{str}

fn str(x) { }  // ERROR: Symbol 'str' already imported from fsl

// Must choose one or the other, or use qualification
```

#### Type-Aware Method Resolution
```frame
import fsl.{set, list}

fn add(a: int, b: int): int { return a + b }

fn main() {
    var mySet = set()
    mySet.add(5)        // Calls Set's add method (type known)
    
    var sum = add(3, 4) // Calls user's add function
    // No conflict - different contexts
}
```

#### Qualified Access Always Available
```frame
import math.utils

fn calculate() { }  // Local function

fn main() {
    calculate()              // Local function
    math.utils.calculate()  // Module function - always works
}
```

### 4. Rust as First-Class Target

#### Complete Feature Mapping
- **Modules**: Direct mapping to Rust's `mod` system
- **Systems**: Struct with impl blocks
- **State Machines**: Enum-based with match expressions
- **FSL**: Distributed as Rust crate `frame_runtime`
- **Ownership**: Inferred from Frame usage patterns
- **Async**: Optional Tokio integration

#### Example Rust Generation
```rust
// From Frame system
system Counter {
    domain: count: int = 0
    
    interface:
        increment(): int
    
    machine:
        $Running {
            increment() {
                self.count += 1
                return self.count
            }
        }
}

// Generates Rust:
pub struct Counter {
    state: CounterState,
    count: i32,
}

enum CounterState {
    Running,
}

impl Counter {
    pub fn new() -> Self {
        Counter {
            state: CounterState::Running,
            count: 0,
        }
    }
    
    pub fn increment(&mut self) -> i32 {
        match self.state {
            CounterState::Running => {
                self.count += 1;
                self.count
            }
        }
    }
}
```

## Breaking Changes

### FSL No Longer Default
**Impact**: All existing code using FSL operations will need updates  
**Migration**: Add `import fsl.{str, int, float, list, map}` to files using FSL

### Strict Symbol Conflicts
**Impact**: Code with duplicate symbols will error  
**Migration**: Rename conflicting symbols or use qualified access

## Migration Guide

### Before (v0.33)
```frame
fn main() {
    var s = str(42)        // FSL available by default
    var nums = list()      // FSL list
    print(s)
}
```

### After (v0.34)
```frame
import fsl.{str, list}     // Must import FSL

fn main() {
    var s = str(42)        // Now works
    var nums = list()      // Now works
    print(s)              // print still available (built-in)
}
```

## Success Metrics

## Risks & Mitigations

### Risk: Large Migration Effort
**Mitigation**: Provide automated migration tool to add FSL imports

### Risk: Confusion from Python Differences  
**Mitigation**: Clear documentation highlighting Frame's enhancements

### Risk: Rust Compilation Complexity
**Mitigation**: Start with core features, add advanced features incrementally

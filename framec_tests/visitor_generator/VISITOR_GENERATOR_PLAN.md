# Visitor Generator Framework Plan

## Overview
The Visitor Generator Framework creates new language visitors for the Frame transpiler using template-based code generation and progressive validation against the Phase 1-8 test suites.

## Architecture

### Core Components

1. **Template Engine** (`templates/`)
   - Language-specific code generation templates
   - Runtime abstraction templates  
   - AST node visitor method templates

2. **Generator Scripts** (`generators/`)
   - Main visitor generator script
   - Language-specific customization scripts
   - Template instantiation logic

3. **Runtime Templates** (`runtime_templates/`)
   - Frame runtime equivalent for each target language
   - Event dispatch mechanisms
   - State management abstractions

4. **Validation Pipeline** (`../common/tests/visitor_generator/`)
   - Progressive test suites (Phase 1-8)
   - Cross-language validation scripts
   - Functional equivalence verification

## Target Languages and Implementation Order

### Phase 1 Implementation Priority
1. **Rust** - Memory-safe systems language (closest to existing architecture)
2. **C** - Low-level imperative language (maximum architectural difference)
3. **C++** - Object-oriented systems language (hybrid paradigm)
4. **C#** - Managed object-oriented language (.NET ecosystem)
5. **Java** - JVM-based object-oriented language (enterprise focus)
6. **Go** - Simple systems language (concurrency-focused)

### Existing (Reference Implementations)
- **Python** - Dynamic interpreted language ✅
- **TypeScript** - Statically-typed JavaScript ✅

## Visitor Generator Process

### Step 1: Language Analysis
```
Input: Target language specification
Output: Language profile (syntax, runtime requirements, paradigm)
```

### Step 2: Template Generation
```
Input: Language profile + AST visitor trait
Output: Language-specific visitor template
```

### Step 3: Runtime Generation  
```
Input: Language profile + Frame runtime requirements
Output: Target language Frame runtime equivalent
```

### Step 4: Progressive Validation
```
Input: Generated visitor + Phase 1-8 tests
Output: Validation report + functional equivalence confirmation
```

## Template Structure

### Visitor Template Format
```rust

// Based on Frame transpiler AST visitor pattern

pub struct {LANGUAGE}Visitor {
    // Core visitor state
    builder: {LANGUAGE}CodeBuilder,
    system_name: String,
    
    // Language-specific state
    {LANGUAGE_SPECIFIC_FIELDS}
}

impl AstVisitor for {LANGUAGE}Visitor {
    // Template-generated visitor methods for each AST node type
    {VISITOR_METHODS}

```

## Generation Algorithm

### 1. Language Profile Creation
```python
def create_language_profile(target_lang):
    return {
        'name': target_lang,
        'paradigm': determine_paradigm(target_lang),
        'syntax_style': analyze_syntax(target_lang),
        'memory_model': analyze_memory_model(target_lang),
        'runtime_requirements': determine_runtime_needs(target_lang),
        'type_system': analyze_type_system(target_lang),
        'compilation_model': determine_compilation(target_lang)
    }
```

### 2. AST Method Mapping
```python
def generate_visitor_methods(language_profile, ast_nodes):
    methods = []
    for node_type in ast_nodes:
        template = get_method_template(language_profile, node_type)
        method = instantiate_template(template, node_type, language_profile)
        methods.append(method)
    return methods
```

### 3. Runtime Adaptation
```python
def generate_runtime(language_profile):
    base_runtime = load_frame_runtime_spec()
    adapted_runtime = adapt_to_language(base_runtime, language_profile)
    return generate_code(adapted_runtime, language_profile)
```

## Validation Strategy

### Progressive Testing
1. **Phase 1**: Basic system, interface methods, simple state machines
2. **Phase 2**: State transitions, event parameters, multiple states  
3. **Phase 3**: Actions and operations blocks
4. **Phase 4**: Domain variables, state variables, hierarchical states
5. **Phase 5**: Advanced expressions, control flow, collections
6. **Phase 6**: Exception handling, async/await, advanced types
7. **Phase 7**: Module system, imports, multi-file projects
8. **Phase 8**: Full language feature equivalence

### Cross-Language Validation
```python
def validate_phase(phase_num, languages):
    results = {}
    for lang in languages:
        transpilation_result = test_transpilation(phase_num, lang)
        execution_result = test_execution(phase_num, lang)
        equivalence_result = test_functional_equivalence(phase_num, lang)
        results[lang] = {
            'transpilation': transpilation_result,
            'execution': execution_result, 
            'equivalence': equivalence_result
        }
    return validate_100_percent_success(results)
```
